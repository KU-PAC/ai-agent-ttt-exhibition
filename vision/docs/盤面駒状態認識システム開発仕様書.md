# **開発仕様書：3×3盤面・駒状態認識システム**

## **1\. システム概要**

カメラから取得した画像を入力とし、3×3の盤面上に配置された駒の状態（赤・青・なし）を判定し、二次元配列として出力する画像処理システム。

* **動作環境:** Python 3.x  
* **主要ライブラリ:** OpenCV (cv2), NumPy (numpy)

## **2\. 全体処理フロー**

1. **入力取得:** カメラからフレーム画像を取得する。  
2. **盤面検出・補正:** 画像内から盤面の4隅を検出し、射影変換（透視変換）を用いて真上から見た正方形の画像に補正・切り出しを行う。  
3. **マス分割:** 補正された画像を縦横3等分し、9つのマス（ROI：Region of Interest）に分割する。  
4. **状態判定:** 各マスをHSV色空間に変換し、赤色・青色のピクセル数をカウントして「赤(表)」「青(裏)」「なし」を判定する。  
5. **出力生成:** 判定結果を3×3の配列データとして返す。

## ---

**3\. 機能詳細仕様**

### **3.1. 盤面の検出と補正**

カメラの角度変動を吸収するため、画像を正規化（正面化）します。開発初期は難易度の低い「緑マーカー方式」で進め、安定したら「外枠検出方式」に移行できるよう、両方の仕様を定義します。

* **盤面の外枠検出**  
  1. 入力をグレースケール化し、適応的閾値処理またはCannyエッジ検出で2値化する。  
  2. cv2.findContours で輪郭を抽出。  
  3. cv2.approxPolyDP で多角形近似を行い、面積が最大となる4角形を盤面とみなす。  
* **射影変換**  
  1. 取得した4つの頂点座標を「左上、右上、右下、左下」の順にソートする。  
  2. 変換後のサイズを定義する（例：WIDTH \= 300, HEIGHT \= 300 px）。  
  3. cv2.getPerspectiveTransform で変換行列を計算し、cv2.warpPerspective で正面からの正方形画像（300x300px）を生成する。

### **3.2. マスの分割 (ROI抽出)**

補正済みの画像を3×3の9マスに分割します。

* **処理:** 300×300pxの画像の場合、100×100pxずつの画像配列（ROIリスト）を作成する。  
* **【重要】ノイズ除去クロップ:**  
  マスの境界にある「黒い区切り線」が判定の邪魔にならないよう、切り出した100x100pxの画像のうち、**中心の80x80px程度だけをさらに切り出して（上下左右に10%程度のマージンを設けて）判定用画像とする。**

### **3.3. 色の判定**

切り出した各マスの中心画像に対して、色の判定を行います。

* **処理:**  
  1. 画像をRGBからHSV色空間に変換 (cv2.cvtColor(img, cv2.COLOR\_BGR2HSV))。  
  2. **青色の検出:** 青のHSV範囲（例: H=100\~130, S=50\~255, V=50\~255）でマスクを作成。  
  3. **赤色の検出:** 【注意】OpenCVのHSVにおいて、**赤色はH=0付近とH=180付近の2つに分かれます。** そのため、H=0\~10のマスクとH=170\~180のマスクを2つ作成し、cv2.bitwise\_or で結合して赤色マスクとする必要があります。  
  4. cv2.countNonZero を用いて、各マスク内の白ピクセル（該当色と判定されたピクセル）の数をカウントする。  
* **判定ロジック:**  
  * 判定閾値（例: ROI全ピクセル数の15%以上）を設定する。  
  * 赤ピクセル数 \> 閾値 ⇒ **「赤 (1)」**  
  * 青ピクセル数 \> 閾値 ⇒ **「青 (2)」**  
  * どちらも閾値以下 ⇒ **「なし (0)」**  
  * ※万が一、両方が閾値を超えた場合は、ピクセル数が多い方を採用する。

**4\. データ入出力定義**

* **入力:** numpy.ndarray （OpenCVのVideoCaptureで取得したフレーム画像）  
* **出力:** list （3×3の二次元配列）  
  Python  
  \# 状態の定義: 0=なし, 1=赤(表), 2=青(裏)  
  board\_state \= \[  
      \[0, 1, 0\],  
      \[2, 1, 2\],  
      \[0, 0, 1\]  
  \]

## **5\. パラメータ調整項目（外部変数化の推奨）**

環境光によって色が変化するため、以下の値はコードの先頭で定数として定義するか、GUI（トラックバーなど）で動的に調整できるようにしておくことを強く推奨します。

1. **各色のHSVの下限・上限値**  
   * RED\_LOWER1, RED\_UPPER1, RED\_LOWER2, RED\_UPPER2  
   * BLUE\_LOWER, BLUE\_UPPER  
   * (緑マーカーを使う場合) GREEN\_LOWER, GREEN\_UPPER  
2. **判定閾値**  
   * 色とみなすための最小ピクセル面積比率（例：0.15）  
3. **クロップマージン**  
   * マス目抽出時に枠線を除外するための余白割合（例：0.1）
